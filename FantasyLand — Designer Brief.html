FantasyLand — Designer Brief

Card Template System · Prepared for the Graphic Designer · April 2026

1. What You're Making

You're designing 4 card templates for an automated graphics system. The system is built in Python and uses your designs as the visual blueprint — it reads your exact specs (colors, fonts, sizes, positions) and generates PNG images automatically every week.

You design once. The system produces hundreds of cards — one per matchup, per team, per week. You never have to manually swap in team names, logos, stats, or colors. The system handles all of that.

The 4 cards are:

  • Card 1 — Defensive Stats Card: shows a team's defensive stats with rank badges
  • Card 2 — Player Log Card: shows recent player performances against a defense
  • Card 3 — Pickle Score Card: shows the Pickles matchup verdict and score breakdown
  • Card 4 — Odds Card: shows betting odds and implied totals for the matchup

3. The Most Important Rule — Dynamic Elements

The team color, team logo, and all data are variables. Every card gets generated for all 32 NFL teams with their own colors and logos — the system already has all of these. Your job is to show WHERE they go and HOW BIG they are. You only need to design one version of each card.

Use Cowboys blue #003594 as the placeholder team color. Wherever a team color appears, label it clearly in your spec sheet: "TEAM PRIMARY COLOR — SWAPPED BY SYSTEM"

Same rule applies to:

  • Team logo — draw a placeholder box at the exact size and position, labeled "TEAM LOGO — PNG"
  • Team name — placeholder text at exact font/size/position
  • All stat values, player names, scores, ranks — placeholder text with exact specs

4. What the System CAN and CANNOT Render

The Python image library draws everything programmatically. It is not a browser — it cannot render CSS-style effects. Every visual effect must be achievable with a flat solid shape.

✅ YES — Fully Supported❌ NO — Will Not Render
Solid fill colorsDrop shadows
Flat rectangles and rounded rectanglesBlur or glow effects
Solid borders / strokesGradient fills
Text at any size, weight, colorTransparency / opacity on text
PNG images pasted flatImage masks or clipping masks on logos
Flat color backgroundsInner shadows
Simple solid color overlaysGradient text

If an effect requires a blur, shadow, or opacity slider in Illustrator — flag it and we'll find a flat alternative.

5. Fonts

Every font must be:

  • Available as a .ttf or .otf file (downloaded to your machine and shared with us alongside the design)
  • A standard weight — no variable fonts
  • Maximum 2 fonts per card (one display/heading, one body/label)

Recommended free fonts (all available at fonts.google.com — download the .ttf files from there):

6. Card Specs

Card 1 — Defensive Stats Card

Artboard: 1000 × 700px (for 4 stat rows — the system stretches for more rows automatically)

Zone 1 — Header (~160px tall)
  • Team color accent bar — full width, ~8px tall, at very top
  • Team logo placeholder — left-aligned, 90×90px, ~20px from left edge, vertically centred in header
  • Eyebrow text — small caps label above team name (e.g. "WR MATCHUP STATS")
  • Team name — large display font
  • Subtitle — small text below team name (e.g. "Opposing Defense · 2025 Season Weeks 1–18")
  • Divider line — full width, at bottom of header zone
Zone 2 — Stat Rows (~80px each, repeating)
  • Left: stat label text (e.g. "Pass Yards Allowed / Game")
  • Right: stat value (large number) + rank badge pill, side by side
  • Rank badge: rounded rectangle pill — 3 color states: GREEN (ranks 1–10), YELLOW (11–20), RED (21–32). Design all 3 versions.
  • Thin divider line between each row
  • Design one row — it repeats. Note the exact height.
Zone 3 — Pickles Strip (~70px tall)
  • Slightly different background shade to the stat rows (darker or subtly different)
  • Left: pickle emoji placeholder + "PICKLES SAYS —" label + verdict text italic (e.g. "Must-Start")
  • Right: score number large (e.g. "8.4") + "/10" small beside it
  • Divider lines above and below the strip
Zone 4 — Footer (~44px tall)
  • Left: season/week text (e.g. "2025 SEASON · WEEKS 1–18")
  • Right: FantasyLand logo placeholder box — ~120×20px
  • Subtle top divider

Card 2 — Player Log Card

Artboard: 1000 × 780px (for 4 player rows)

Zone 1 — Header (~150px) — Same as Card 1
  • Team color accent bar, logo, eyebrow, team name, subtitle
  • Subtitle here: "Recent WR Performances Against This Defense"
Zone 2 — Section Strip (~34px)
  • Slightly darker band, full width
  • Small label text: e.g. "LAST 4 WEEKS · PPR SCORING"
Zone 3 — Player Rows (~110px each, repeating)
  • Left side: week label small top (e.g. "WEEK 18") → player name large bold → stat line small below (e.g. "9 REC · 122 YDs · 1 TD")
  • Right side: fantasy points number — very large, this is the hero number per row → "FPTS" label tiny below it
  • Points color: GREEN (20+ pts), YELLOW (10–19.9 pts), RED (under 10 pts)
  • Thin divider between rows
  • Design one row — it repeats. Note the exact height.
Zone 4 — Pickles Strip — identical to Card 1
Zone 5 — Footer — identical to Card 1

Card 3 — Pickle Score Card

Artboard: 1000 × 820px

This card is standalone — it's about Pickles' matchup verdict, not a team's defense. More breathing room, the score number is the centrepiece. Centred layout throughout.

Zone 1 — Pickles Header (~130px)
  • Centred layout — no team logo here
  • Pickle emoji placeholder + "Pickles" name in italic display font, centred
  • Subtle divider below
Zone 2 — Matchup Context (~100px)
  • Two team logo placeholders side by side, centred, with "AT" or "VS" text between them
  • Each logo: 48×48px
  • Small text below: e.g. "LA Rams at Dallas Cowboys · WR · 2025 Week 18"
  • Divider below
Zone 3 — Verdict Area (~220px)
  • Small label centred: "PICKLES SAYS —"
  • Verdict stamp: rounded rectangle box, team color border + light color background tint inside, verdict text large bold inside (e.g. "MUST-START") — stamp rotated slightly (~2°)
  • Design the stamp in green — the system swaps border/fill/text color based on verdict (green = must-start, yellow = caution, red = sit him)
Zone 4 — Score Number (~120px)
  • Hero number centred, very large (e.g. "8.4") — biggest text on the card
  • Score bar below: full-width horizontal track, filled portion in verdict color
  • Small labels at each end of the bar: "1 — SIT HIM" left, "10 — MUST-START" right
Zone 5 — Breakdown Row (~100px)
  • 3 equal columns separated by vertical dividers
  • Each column: label (DEFENSE / GAME TOTAL / IMPLIED TOTAL) → sub-score number → weight label ("50% weight" etc.)
  • Horizontal divider above entire row
Zone 6 — Footer — identical to Cards 1 & 2

9. Deliverable Checklist

  • 4 Illustrator artboards (one per card) — in one file or separate files
  • Spec sheet artboard alongside each card with all values filled in
  • All font .ttf or .otf files in a folder
  • No effects that can't be rendered flat (no shadows, blurs, glows, gradients)
  • Every team-color element labeled "TEAM PRIMARY COLOR — SWAPPED BY SYSTEM"
  • Every placeholder labeled with what it represents
  • Exact padding, row height, radius, font size, and hex values all documented
  • All 3 rank badge color states designed (green / yellow / red)
